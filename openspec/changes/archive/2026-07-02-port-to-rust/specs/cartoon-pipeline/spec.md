## 新增需求

### 需求：调整 RGB 图像的饱和度

系统应使用灰度值与原始色彩之间的线性插值来调整图像饱和度。
公式：`result = (1 - scalar) * gray + scalar * color`，其中 `gray = 0.299*R + 0.587*G + 0.114*B`。

#### 场景：scalar=1 时不变
- **当** 饱和度系数为 1.0
- **则** 输出像素应在浮点容差内等于输入像素

#### 场景：scalar=0 时为灰度
- **当** 饱和度系数为 0
- **则** 每个输出像素的三个通道应全部相等（R=G=B）

#### 场景：scalar=2 时饱和度增强
- **当** 饱和度系数为 2.0
- **则** 逐像素通道方差应大于或等于输入方差

#### 场景：输出值钳制
- **当** 饱和度调整产生超出 [0.0, 1.0] 的值
- **则** 输出应被钳制到 [0.0, 1.0]

### 需求：应用双边滤波以平滑图像同时保留边缘

系统应使用可配置的窗口半径、空域标准差和色域标准差应用双边滤波。
对于彩色图像，滤波应在 CIELab 色彩空间中执行，色域标准差乘以 100。
滤波器应支持多次迭代。

#### 场景：窗口半径为零时为恒等变换
- **当** 以 radius=0 调用双边滤波
- **则** 输出应在浮点容差内等于输入

#### 场景：极大色域标准差近似高斯模糊
- **当** 色域标准差设为极大值（如 1000）
- **则** 边缘应被平滑，效果类似相同空域标准差的高斯模糊

#### 场景：输出尺寸保持
- **当** 对任意有效图像应用双边滤波
- **则** 输出宽和高应等于输入宽和高

#### 场景：多次迭代复合平滑
- **当** loop_num 为 3
- **则** 双边滤波应恰好按顺序执行 3 次

#### 场景：并行执行结果一致
- **当** 双边滤波以不同线程数运行
- **则** 像素值应在浮点容差内完全一致

### 需求：使用 Sobel 算子检测边缘

系统应使用 3×3 Sobel 核和可配置阈值在灰度图像中检测边缘。

#### 场景：均匀图像无边
- **当** 处理均匀灰度图像（所有像素相等）
- **则** 不应检测到任何边缘像素（所有掩码值 = 0）

#### 场景：清晰渐变边界处有边缘
- **当** 处理一个已知渐变的合成图像
- **则** 在适当阈值下，边缘像素应出现在边界位置

#### 场景：较低阈值产生更多边缘
- **当** 在同一图像上阈值从 0.1 降至 0.01
- **则** 检测到的边缘像素数量应增加

### 需求：将边缘掩码叠为黑色描边

系统应将边缘掩码叠加到模糊图像上，将边缘像素变为黑色（RGB=0,0,0）。

#### 场景：全为边的掩码生成全黑图像
- **当** 边缘掩码全为 1.0
- **则** 所有输出像素应为 (0.0, 0.0, 0.0)

#### 场景：无边掩码保留原图
- **当** 边缘掩码全为 0.0
- **则** 输出应在浮点容差内等于模糊输入

#### 场景：混合掩码正确合成
- **当** 部分像素为边、部分为非边
- **则** 边缘像素应为黑色、非边缘像素应保留模糊值

### 需求：端到端卡通化管线

系统应提供 `cartoonize` 函数执行完整管线：
饱和度调整 → 双边滤波 → 边缘检测 → 边缘叠加，接受所有可配置参数。

#### 场景：默认参数产生有效输出
- **当** 以默认参数处理 `test.jpg`
- **则** 输出应与输入尺寸相同，且所有值在 [0.0, 1.0] 内

#### 场景：管线返回处理元数据
- **当** 卡通化完成
- **则** 函数应返回各管线阶段的耗时和详情信息
