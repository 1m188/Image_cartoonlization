# cli-interface

## Purpose

命令行界面，解析参数、校验范围、提供详细模式输出，并根据文件扩展名决定输出格式。各语言移植版可使用各自惯用的参数解析方案（Rust→clap，Go→flag，C#→手动解析）。

## Requirements

### 需求：使用 clap 解析命令行参数

系统应接受与 Go 移植版 CLI 界面一致的所有参数。
必选：`-i <路径>`。可选：`-o <路径>`、`--edge-thresh`、`--sat`、`--radius`、`--sigma-d`、`--sigma-r`、`--loop`、`--workers`、`-v`、`-h`/`--help`。

#### 场景：帮助标志显示用法
- **当** 传入 `--help`
- **则** 应显示帮助文本，退出码为 0

#### 场景：短帮助标志与长标志行为一致
- **当** 传入 `-h` 短标志
- **则** 行为应与 `--help` 完全一致

#### 场景：缺少输入路径报错
- **当** 未提供 `-i` 参数
- **则** 程序应打印错误信息并以非零退出码退出

#### 场景：接受所有参数
- **当** 所有可选标志均以有效值提供
- **则** 程序应无错误地解析它们

### 需求：校验参数范围

系统应校验所有数值参数是否在允许范围内，并对违规情况打印清晰的错误信息。

#### 场景：拒绝越界的 edge-thresh
- **当** `--edge-thresh` 为 1.5
- **则** 程序应打印错误并以非零退出码退出

#### 场景：拒绝 radius 为零
- **当** `--radius` 为 0
- **则** 程序应打印错误并以非零退出码退出

#### 场景：拒绝负数 sigma
- **当** `--sigma-d` 为 -1
- **则** 程序应打印错误并以非零退出码退出

#### 场景：拒绝 loop 越界
- **当** `--loop` 为 15（超出 [1, 10] 范围）
- **则** 程序应打印错误并以非零退出码退出

#### 场景：拒绝 -i 后跟标志类值
- **当** `-i` 后跟以 "-" 开头的值（如 `-i --edge-thresh`）
- **则** 程序应打印明确错误提示并退出非零

#### 场景：拒绝数值参数缺值
- **当** 数值参数（如 `--edge-thresh`）后缺少值
- **则** 程序应打印明确错误（如"缺少值"）并退出非零

#### 场景：拒绝 -o 缺值
- **当** `-o` 后缺少输出路径值
- **则** 程序应打印明确错误并退出非零

#### 场景：多标志正确跳过已消耗的值
- **当** 连续使用多个带值的可选参数（如 `--radius 5 --sat 3.0`）
- **则** 程序应正确解析，不将前一个参数的值误解析为后一个参数的标志

### 需求：详细模式输出每阶段耗时

系统应在设置 `-v` 标志时打印每阶段处理时间和详细信息。

#### 场景：在 test.jpg 上的详细输出
- **当** 以有效输入传入 `-v`
- **则** 程序应打印饱和度调整、双边滤波、边缘检测、边缘叠加和总耗时等行

#### 场景：非详细模式成功时静默
- **当** 未传入 `-v`
- **则** 程序应仅打印总耗时

### 需求：根据文件扩展名决定输出格式

当扩展名为 `.jpg` 或 `.jpeg` 时系统应以 JPEG 保存输出，当扩展名为 `.png` 时以 PNG 保存。

#### 场景：输出到 .jpg 生成 JPEG
- **当** 输出路径以 `.jpg` 结尾
- **则** 输出文件应为合法 JPEG

#### 场景：不支持的扩展名报错
- **当** 输出路径以 `.bmp` 结尾
- **则** 程序应打印错误并以非零退出码退出

### 需求：与 Go/Rust 移植版行为一致

在相同的输入图片和参数下，各语言移植版（Go/Rust/C#）的输出应视觉上一致（允许在 uint8 量化误差范围内的浮点差异）。

#### 场景：相同参数产生相同输出尺寸
- **当** 以默认参数在三个端口（Go/Rust/C#）中处理 `test.jpg`
- **则** 输出尺寸应完全一致
