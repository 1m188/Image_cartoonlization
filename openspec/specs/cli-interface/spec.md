# cli-interface

## Purpose

命令行界面，使用 clap 解析参数、校验范围、提供详细模式输出，并根据文件扩展名决定输出格式。

## Requirements

### 需求：使用 clap 解析命令行参数

系统应接受与 Go 移植版 CLI 界面一致的所有参数。
必选：`-i <路径>`。可选：`-o <路径>`、`--edge-thresh`、`--sat`、`--radius`、`--sigma-d`、`--sigma-r`、`--loop`、`--workers`、`-v`、`-h`/`--help`。

#### 场景：帮助标志显示用法
- **当** 传入 `--help`
- **则** 应显示帮助文本，退出码为 0

#### 场景：缺少输入路径报错
- **当** 未提供 `-i` 参数
- **则** 程序应打印错误信息并以非零退出码退出

#### 场景：接受所有参数
- **当** 所有可选标志均以有效值提供
- **则** 程序应无错误地解析它们

### 需求：校验参数范围

系统应校验所有数值参数是否在允许范围内，并对违规情况打印清晰的错误信息。

#### 场景：拒绝越界的 edge-thresh
- **当** `--edge-thresh` 为 1.5
- **则** 程序应打印错误并以非零退出码退出

#### 场景：拒绝 radius 为零
- **当** `--radius` 为 0
- **则** 程序应打印错误并以非零退出码退出

#### 场景：拒绝负数 sigma
- **当** `--sigma-d` 为 -1
- **则** 程序应打印错误并以非零退出码退出

### 需求：详细模式输出每阶段耗时

系统应在设置 `-v` 标志时打印每阶段处理时间和详细信息。

#### 场景：在 test.jpg 上的详细输出
- **当** 以有效输入传入 `-v`
- **则** 程序应打印饱和度调整、双边滤波、边缘检测、边缘叠加和总耗时等行

#### 场景：非详细模式成功时静默
- **当** 未传入 `-v`
- **则** 程序应仅打印总耗时

### 需求：根据文件扩展名决定输出格式

当扩展名为 `.jpg` 或 `.jpeg` 时系统应以 JPEG 保存输出，当扩展名为 `.png` 时以 PNG 保存。

#### 场景：输出到 .jpg 生成 JPEG
- **当** 输出路径以 `.jpg` 结尾
- **则** 输出文件应为合法 JPEG

#### 场景：不支持的扩展名报错
- **当** 输出路径以 `.bmp` 结尾
- **则** 程序应打印错误并以非零退出码退出
