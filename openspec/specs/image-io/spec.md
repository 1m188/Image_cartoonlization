# image-io

## Purpose

从 JPEG 或 PNG 文件加载图像到内部表示，以及将图像保存为 JPEG 或 PNG 文件。

## Requirements

### 需求：从 JPEG 或 PNG 文件加载图像

系统应将 JPEG 和 PNG 图像文件加载为内部 `ImageData` 表示，
像素值归一化为 [0.0, 1.0] 的 float64。

#### 场景：加载合法的 JPEG 文件
- **当** 加载 `test.jpg`（2812×1280 JPEG）
- **则** 返回的 ImageData 应宽=2812、高=1280、通道=3

#### 场景：加载合法的 PNG 文件
- **当** 加载合法的 PNG 文件
- **则** 返回的 ImageData 应具有正确的尺寸和通道数

#### 场景：拒绝不支持的格式
- **当** 加载 GIF 文件
- **则** 函数应返回错误

#### 场景：拒绝不存在的文件
- **当** 加载不存在的路径
- **则** 函数应返回错误

### 需求：将图像保存为 JPEG 或 PNG 文件

系统应将 `ImageData` 表示保存为 JPEG 或 PNG 文件。
JPEG 输出应使用质量 95。格式由文件扩展名决定。

#### 场景：另存为 JPEG 生成合法 JPEG
- **当** 将 ImageData 保存到 `.jpg` 路径
- **则** 生成的文件应为可重新加载的合法 JPEG

#### 场景：另存为 PNG 生成合法 PNG
- **当** 将 ImageData 保存到 `.png` 路径
- **则** 生成的文件应为可重新加载的合法 PNG

#### 场景：保存后重新加载往返
- **当** 保存图像后重新加载
- **则** 重新加载的像素应在 uint8 量化误差内匹配原始值

#### 场景：拒绝不支持的输出扩展名
- **当** 保存到 `.bmp` 路径
- **则** 函数应返回错误
